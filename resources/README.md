# Reference data

External data the workflow needs. Fetch once, then point `config.yaml` at it.

## KEGG module definitions (required for the module layer)
```bash
python resources/fetch_kegg_modules.py --out resources/kegg_modules.tsv
```
Writes `module_id, name, definition`. Uses the public KEGG REST API (rate-
limited; the script sleeps and is resumable). Set `references.kegg_module_def`.

## GTDB reference trees + taxonomy (required for phylogeny)
Download the GTDB release used for your taxonomy (the metadata is GTDB-based).
Set in `config.yaml phylogeny:`
- `bac120_tree`, `ar53_tree` — reference trees (Newick).
- `bac120_taxonomy`, `ar53_taxonomy` — `accession <tab> taxonomy` tables.
From https://data.gtdb.ecogenomic.org/releases/ (use the release matching your
annotations). Tips are species-representative accessions; the pipeline maps your
species to them.

## GTDB-Tk database (only if grafting unplaced species)
Needed when `phylogeny.unplaced_species: gtdbtk_denovo`. Set `phylogeny.gtdbtk_db`.

## Annotation tool databases (for de novo layers)
- dbCAN database: `references.dbcan_db` (run `dbcan_build` / download per dbCAN docs).
- AMRFinderPlus database: `references.amrfinder_db` (`amrfinder -u`).
- antiSMASH databases: `references.antismash_db` (`download-antismash-databases`).

These three layers (CAZyme/BGC/AMR) are generated by the workflow because they
were not in the provided Prokka/eggNOG/KofamScan set. KO, Pfam, COG, EC come
from your existing eggNOG/KofamScan outputs and need no extra database.
